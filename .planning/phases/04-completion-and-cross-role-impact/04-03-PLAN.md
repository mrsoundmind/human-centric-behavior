Create `src/components/experiences/internal/designation-portal/CrossRoleImpactView.tsx`:

**Props interface:**
```typescript
interface CrossRoleImpactViewProps {
  decisions: DecisionRecord[];
}
```

**Component logic:**

1. **Build choiceId set from user's decisions:**
   ```typescript
   const choiceIds = useMemo(() => new Set(decisions.map(d => d.choiceId)), [decisions]);
   ```

2. **Filter crossRoleImpactMap for user's actual choices:**
   ```typescript
   const relevantEdges = useMemo(
     () => crossRoleImpactMap.filter(e => choiceIds.has(e.sourceChoiceId)),
     [choiceIds]
   );
   ```

3. **Return null if no impacts** — do not render an empty section, just return null.

4. **Group edges by targetRole:**
   ```typescript
   const groupedByRole = useMemo(() => {
     const groups = new Map<Designation, ImpactEdge[]>();
     for (const edge of relevantEdges) {
       const existing = groups.get(edge.targetRole) ?? [];
       existing.push(edge);
       groups.set(edge.targetRole, existing);
     }
     return groups;
   }, [relevantEdges]);
   ```

5. **Define severity color config:**
   ```typescript
   const SEVERITY_CONFIG = {
     high:   { label: "High Impact",   className: "border-red-500/40 bg-red-500/10 text-red-300" },
     medium: { label: "Medium Impact", className: "border-amber-500/40 bg-amber-500/10 text-amber-300" },
     low:    { label: "Low Impact",    className: "border-blue-500/40 bg-blue-500/10 text-blue-300" },
   } as const;
   ```

6. **Define role display names:**
   ```typescript
   const ROLE_DISPLAY: Record<Designation, string> = {
     sales: "Sales", pm: "Product Manager", developer: "Developer",
     qa: "QA", designer: "Designer", ba: "Business Analyst",
     crm: "CRM", strategy: "Strategy",
   };
   ```

7. **Render structure:**
   - Section wrapper: `motion.div` with `initial={{ opacity: 0, y: 20 }}`, `animate={{ opacity: 1, y: 0 }}`
   - Section label: `text-xs font-mono text-gray-500 uppercase tracking-widest mb-6` — "HOW YOUR DECISIONS RIPPLED"
   - For each `[targetRole, edges]` in `groupedByRole`:
     - Role heading: `text-lg font-semibold text-white mb-3` — `ROLE_DISPLAY[targetRole]`
     - For each edge in edges:
       - Card: `bg-white/[0.02] border border-white/[0.05] rounded-xl p-5 mb-3`
       - Phase tag: `text-xs font-mono text-gray-500 uppercase tracking-widest` — `edge.targetPhase`
       - Consequence text: `text-gray-300 leading-relaxed mt-2` — `edge.consequence`
       - Severity badge: `text-xs px-2 py-0.5 rounded-full inline-block mt-2` with `SEVERITY_CONFIG[edge.severity].className` and label
   - Framer Motion stagger on cards: each card gets `transition={{ delay: index * 0.05 }}`

**Tone:** The consequence strings should read as empathy-building ("Your choice created this consequence for..."), not blame. The data itself carries the right framing — it comes from the scenario `crossRoleImpact[].description` which was authored with this intent in Phase 3.

**Critical rules:**
- Only show impacts from choices the user actually made — choiceId filtering is mandatory
- Do NOT render anything if relevantEdges is empty — return null
- Use dark theme cards matching the rest of the completion screen
- Import crossRoleImpactMap from the data layer, not from a React component
