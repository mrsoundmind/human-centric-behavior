
2. Add private helper `detectShifts(phaseScores: PhaseScore[]): BehavioralShift[]`:
   - Iterate consecutive phase pairs in the PhaseScore array
   - For each pair, get dominant dimension of each phase using the existing `dominantDimension` helper
   - If dominant dimensions differ AND both phases have decisionCount >= MIN_DECISIONS_PER_PHASE (2), record a BehavioralShift
   - Build the `description` string as: `"Your ${fromDimension} pattern in ${fromPhase} gave way to ${toDimension} in ${toPhase}"`
   - Replace underscores with spaces in dimension names for readable descriptions (e.g., "conflict_avoidance" becomes "conflict avoidance")

3. Add private helper `findDominantPhase(phaseScores: PhaseScore[], dimension: FrictionDimension): SDLCPhase`:
   - Among phases with decisionCount >= MIN_DECISIONS_PER_PHASE (2), find the phase where the given dimension has the highest percentage
   - If no phase meets the minimum, fall back to the phase with the highest raw percentage for that dimension
   - On tie, use first-encountered phase (earlier in the array)

4. Add private helper `isBalanced(phaseScores: PhaseScore[]): boolean`:
   - Compute overall dimension percentages across all phases (total count per dimension / total decisions)
   - Return true if no dimension exceeds BALANCED_THRESHOLD (0.4)

5. Add private helper `findSecondaryDimension(phaseScores: PhaseScore[], primary: FrictionDimension): FrictionDimension | null`:
   - Compute overall dimension percentages across all phases
   - Return the dimension with the second-highest percentage, excluding the primary
   - Return null if the secondary is less than 15% of total decisions (too insignificant)
   - On tie, use DIMENSION_TIEBREAK_ORDER

6. Export `computeProfile(decisions: DecisionRecord[]): ProfileResult`:
   - If decisions.length < MIN_DECISIONS_FOR_PROFILE (3), return `{ status: "insufficient_data", decisionCount: decisions.length }`
   - Call `computeRunningScore(decisions)` to get phaseScores
   - If `isBalanced(phaseScores)`, use BALANCED_ARCHETYPE and set dominantDimension to the highest (via tiebreak), dominantPhase to the first phase
   - Otherwise:
     a. Compute overall dominant dimension across all phases (total count per dimension / total decisions, with DIMENSION_TIEBREAK_ORDER for ties)
     b. Find dominant phase for that dimension using `findDominantPhase`
     c. Look up archetype: try `ARCHETYPE_LOOKUP[\`${dominantDimension}:${dominantPhase}\`]` first, fall back to `DIMENSION_FALLBACK[dominantDimension]`
     d. Find secondary dimension via `findSecondaryDimension`
     e. Detect shifts via `detectShifts(phaseScores)`
   - Construct and return `{ status: "complete", profile: ScoringProfile }` with all fields from the archetype definition plus computed breakdown, shifts, and dimension info

CRITICAL constraints:
- Do NOT modify existing computeRunningScore or dominantDimension functions
- Do NOT add any React imports
- Do NOT add any "totalScore", "grade", or "points" fields
- The function must be pure: no side effects, no mutation of input
