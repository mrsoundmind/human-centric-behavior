**File 1: `data/scenarios/schema.ts`** — Zod schemas as the SINGLE SOURCE OF TRUTH.

Define these Zod schemas in this exact order:
1. `FrictionDimensionSchema` = `z.enum(["client_friction", "internal_bureaucracy", "knowledge_gap", "conflict_avoidance"])`
2. `SDLCPhaseSchema` = `z.enum(["discovery", "requirements", "design", "development", "testing", "launch", "maintenance"])`
3. `DesignationSchema` = `z.enum(["sales", "pm", "developer", "qa", "designer", "ba", "crm", "strategy"])`
4. `CrossRoleImpactSchema` = `z.object({ affectedRole: DesignationSchema, affectedPhase: SDLCPhaseSchema, description: z.string().min(1), severity: z.enum(["low", "medium", "high"]) })`
5. `ChoiceSchema` = `z.object({ id: z.string().min(1), text: z.string().min(1), frictionTag: FrictionDimensionSchema, crossRoleImpact: z.array(CrossRoleImpactSchema) })` — crossRoleImpact is required array (empty allowed, but field must exist)
6. `DebriefSchema` = `z.object({ whatHappened: z.string().min(1), why: z.string().min(1), how: z.string().min(1), whoBecame: z.string().min(1), tomorrowAction: z.string().min(1) })` — all 5 fields required non-empty
7. `ScenarioConfigSchema` = `z.object({ id: z.string().min(1), role: DesignationSchema, phase: SDLCPhaseSchema, frictionDimension: FrictionDimensionSchema, title: z.string().min(1), setup: z.string().min(1), choices: z.array(ChoiceSchema).min(2), debrief: DebriefSchema })`

Export `validateScenario(raw: unknown, filePath?: string): ScenarioConfig` function using `safeParse` — on failure, throw Error with file path and formatted Zod error. On success, return `result.data`.

Export all schemas as named exports.

**File 2: `data/scenarios/types.ts`** — TypeScript types DERIVED from Zod schemas.

```typescript
import type { z } from "zod";
import type {
  FrictionDimensionSchema,
  SDLCPhaseSchema,
  DesignationSchema,
  CrossRoleImpactSchema,
  ChoiceSchema,
  DebriefSchema,
  ScenarioConfigSchema,
} from "./schema";

export type FrictionDimension = z.infer<typeof FrictionDimensionSchema>;
export type SDLCPhase = z.infer<typeof SDLCPhaseSchema>;
export type Designation = z.infer<typeof DesignationSchema>;
export type CrossRoleImpact = z.infer<typeof CrossRoleImpactSchema>;
export type Choice = z.infer<typeof ChoiceSchema>;
export type Debrief = z.infer<typeof DebriefSchema>;
export type ScenarioConfig = z.infer<typeof ScenarioConfigSchema>;
```

Also define `DecisionRecord` interface in types.ts (not Zod-derived — runtime-only):
```typescript
export interface DecisionRecord {
  choiceId: string;
  frictionTag: FrictionDimension;
  timestamp: number;
  phase: SDLCPhase;
  role: Designation;
}
```

Note: `SDLCPhase` is defined here in `data/scenarios/types.ts` as the canonical type — NOT imported from SDLCContext.tsx. Per RESEARCH.md Open Question 1, this is new infrastructure independent of the existing Layer 2 phases.

Note: The existing `Designation` type in `DesignationSelect.tsx` has the same 8 values. The new type in `types.ts` is the canonical version for the designation portal. They are compatible but intentionally separate — the existing one serves the UI, this one serves the data layer.
